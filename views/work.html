Software Engineer, Microsoft

As a software engineer at Azure Automation, I worked on migrating PowerShell and Python scripts from a bulky VM-based setup to a sleek, containerized microservices architecture using .NET and Docker. This migration made things more secure by running scripts in isolated Hyper-V containers, improving our success rate from 99.8% to 99.9% for over 200M executions each month.

I built internal APIs and developed custom PowerShell cmdlets and Python packages to let customers easily access their Azure assets, like credentials and certificates.

I also worked on debugging and deploying microservices to support communication between containers and servers, which helped us scale to new regions. By automating monitoring and deployment scripts, I helped cut down on manual work for expanding into additional data centers.

Codeforces Rating Updates

Developed an application that automates the delivery of Codeforces contest rating updates. The project uses Google Firebase to manage user data and AWS EC2 to execute Python scripts that handle the updates. I built the user interface with HTML, CSS, and JavaScript and hosted it on GitHub Pages.

Once a contest's ratings are published, the application sends an email notification to users, saving them the hassle of manually checking the Codeforces website. This project made it easier for users to stay updated on their contest performance. I'm currently working on releasing this to production so that contestants can start using it.

Cache Simulator

As part of the Computer Systems course at CMU, I worked on creating a cache simulator from scratch in C. The simulator implemented an LRU (Least Recently Used) cache algorithm, taking the number of sets, blocks, and associativity (E) as input. It then processed a series of load and store instructions, calculating hits, misses, dirty bytes, evictions, and dirty bytes evicted. This project gave me hands-on experience with cache management and performance optimization.
